---
name: scope-check
description: Check if a proposed feature belongs in this project
---

Evaluate whether a proposed feature or change fits the project scope.

1. Read DESIGN.md §2 (Recommended Scope) and §7 (Anti-Bloat Guardrails).
2. Evaluate the proposed feature against:
   - Is it in the Must Have, Nice to Have, or Exclude list?
   - Does it directly improve search recall or usage pattern understanding?
   - Would it pass the anti-bloat guardrails?
3. Give a clear verdict: build it, defer it, or reject it — with reasoning.
